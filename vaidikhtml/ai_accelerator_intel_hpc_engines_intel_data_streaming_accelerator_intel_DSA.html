Realize Faster Time to Data Insights for Better Business Outcomes

Accelerate data movement to reduce time to data insights and make better business decisions—faster—with Intel® Data Streaming Accelerator (Intel® DSA).

This integrated accelerator on Intel® Xeon® Scalable processors accelerates workloads by offloading data movement from the processor, allowing the CPU to perform other advanced analytics-related tasks. Intel® DSA comes complete with tools and implementation resources so developers and architects can more easily and quickly perform data movement and transformation operations in both new and existing workloads—driving improved outcomes with reduced costs.

Lower TCO and Enhance Sustainability

Intel® DSA not only increases performance per watt to help organizations meet their sustainability goals but also allows businesses to optimize workloads so fewer servers are needed for the same performance envelope, helping to reduce TCO significantly.

Enable High Performance

Intel® DSA comes with resources to accelerate performance, including Intel® Data Mover Library (Intel® DML), an open source library that improves the performance of databases, communications, and applications. By using Intel® DML with Intel® DSA, analysts and system admins can access data faster with higher input/output operations per second (IOPS) with NVMe.

Streamline Integration

As a built-in accelerator in Intel® Xeon® Scalable processors, Intel® DSA simplifies integration by eliminating the need for specialized hardware, additional vendors, and disparate support. We also provide developers with consolidated access to tools and resources to further streamline implementation.

Discover Top Intel® Data Streaming Accelerator Use Cases

Intel® DSA helps optimize data movement and transformation operations for demanding workloads. See the different use cases Intel® DSA supports across the data center, storage, networking, and virtual machines (VMs).

Data Center

Offload data movement to reduce the data center burden when performing memory copying or zeroing and free up CPU cycles for advanced analytics workloads.

Storage

Use for data movement in storage appliances using non-transparent bridging (NTB), cyclic redundancy check (CRC) generation, and DIF generation, with or without simultaneously moving data.

Networking

Copy data in packet-processing pipelines. For example, use as a virtual switch offload for inter-VM packet switching.

Deduplication

Compare memory pages for parity to support deduplication.

VM Migration and Fast Checkpointing

Use when a VM manager (VMM) must identify a VM’s modified pages and efficiently send them to a destination machine.

Data Movement between Peer Devices

Optimize data movement between a peer accelerator device and host memory or between two peer devices to free up CPU cycles.

Data Movement to/from/between VMs

Free up CPU cores from performing routine infrastructure tasks such as moving data between VMs, containers, and bare-metal hosts.
