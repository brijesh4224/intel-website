Parallel Programming Techniques for Extracting Rectilinear Images

Doug_B_Intel

03-18-2024

The fourth blog in a series about 360-degree images discusses parallel programming techniques, which are part of the open industry standards called oneAPI, managed by the Unified Acceleration Foundation. The blog explains how dividing coding tasks into independent work items that can be computed in parallel can improve throughput, especially in modern hardware with multiple compute elements in both CPUs and GPUs. The techniques are applied to a use case of extracting a rectilinear image from a 360-degree image, but they can be generalized to many other use cases. The shift to parallel programming constructs can lead to a 1000 to 1700-time improvement over the initial implementation. However, the results can vary depending on several factors, including the efficiency of the initial algorithm, the ability to sub-divide the algorithm into independent computational pieces, the required precision, and the capabilities of the hardware platform.

Optimization Techniques for Extracting Rectilinear Images

Doug_B_Intel

03-11-2024

This blog series discusses 360-degree videos and imaging, focusing on open-source solutions for 360-degree projection. The previous posts noted that while these solutions effectively extract images from the equirectangular format, they are time-consuming. Frame rates varied from 1 to 12 FPS, with the ideal rate depending on the application. The third blog introduces a new open-source development that provides a framework for testing algorithm implementations and timing code parts to determine FPS. The blog also introduces two tools, Intel® VTune and Intel® Advisor, used to identify code hotspots for optimization. Despite using serial programming techniques, the final implementation achieved a significant improvement, reaching 170 FPS compared to the initial 1 FPS.
